GitHub About:
`Browser-first llama.cpp model manager with CLI, desktop launcher, GGUF discovery, validated runtime selection, local build bootstrap, and OpenAI-compatible local endpoint support.`

Short repo blurb:

`llama-model-manager` is a portable control surface for local `llama.cpp` deployments. It gives users a clean browser dashboard, a simple CLI, and lightweight desktop launchers for switching models, tuning defaults, discovering GGUFs, checking live server health, and compiling host-specific runtimes when a safe binary is not already available. The package stays user-space friendly, includes public screenshots, a demo GIF, and a social preview card, and works well with local OpenAI-compatible harnesses such as `opencode`.

Expanded repo intro:

`llama-model-manager` turns a raw local `llama.cpp` setup into something people can actually operate. Users can switch between registered models, scan for new GGUFs, edit per-model presets, inspect live runtime health, and manage single-client or multi-client posture from one place. The package ships with a browser-first dashboard, CLI tools, desktop entry, example configs, sanitized public assets, and a demo mode for clean documentation captures.
